## Endpoint

```
POST /api/v1/links
```

## Headers

| Header | Required | Notes |
|---|---|---|
| `Authorization: Bearer <token>` | Yes | JWT or `lk_live_` API key |
| `x-workspace-id: <uuid_or_slug>` | Yes (JWT) / Optional (API key) | Inferred from key if omitted; must match key's workspace if included |
| `Content-Type: application/json` | Yes | |

## Body

<ParamField body="url" type="string" required>
  Destination URL. Must include `https://` or `http://`.
</ParamField>

<ParamField body="title" type="string">
  Internal label. Not visible to visitors.
</ParamField>

<ParamField body="description" type="string">
  Internal context.
</ParamField>

<ParamField body="customSlug" type="string">
  Custom short code. Recommended character set: lowercase letters, digits, hyphens (no server-side regex enforcement, but stick to this for portability and readability). If omitted, a 7-char nanoid is generated.
</ParamField>

<ParamField body="utmSource" type="string">UTM source, e.g., `google`.</ParamField>
<ParamField body="utmMedium" type="string">UTM medium, e.g., `cpc`.</ParamField>
<ParamField body="utmCampaign" type="string">UTM campaign, e.g., `q2_launch`.</ParamField>
<ParamField body="utmTerm" type="string">UTM term (paid search keyword).</ParamField>
<ParamField body="utmContent" type="string">UTM content (creative differentiator).</ParamField>

<ParamField body="customUtmParams" type="object">
  Arbitrary custom UTM keys. `{"audience": "lookalike_3"}` becomes `&utm_audience=lookalike_3` on redirect.
</ParamField>

<ParamField body="templateId" type="string">
  UUID of a UTM template to apply. Pre-fills UTM fields from the saved template. Any UTM fields you send explicitly take precedence.
</ParamField>

<ParamField body="domainId" type="string">
  UUID of the workspace's domain to use. Defaults to the workspace's default domain. Must be verified and enabled.
</ParamField>

<ParamField body="folderId" type="string">
  UUID of the folder to place the link in. Pass `null` for unfiled.
</ParamField>

<ParamField body="tagIds" type="string[]">
  Array of tag UUIDs to attach.
</ParamField>

<ParamField body="password" type="string">
  Stored on the link record. Counts against the workspace's password-protected link maximum. The public redirect does not currently gate on this - see [Password-protected links](/links/password-protected).
</ParamField>

<ParamField body="expiresAt" type="string">
  ISO 8601 timestamp. After this, redirect returns `410 Gone`.
</ParamField>

<ParamField body="clickLimit" type="integer">
  Max clicks. Recorded for reporting; the public redirect does not currently stop on it. Pair with `expiresAt` for a guaranteed cutoff.
</ParamField>

<ParamField body="iosUrl" type="string">
  Stored as the intended destination for iOS clicks. The public redirect does not currently route based on user-agent - verify behavior before relying on this.
</ParamField>

<ParamField body="androidUrl" type="string">
  Same caveat as `iosUrl`.
</ParamField>

<ParamField body="geoTargets" type="object">
  Stored as a country-code → URL map. The public redirect does not currently route based on country.
</ParamField>

<ParamField body="ogTitle" type="string">Override OG title for social previews.</ParamField>
<ParamField body="ogDescription" type="string">Override OG description.</ParamField>
<ParamField body="ogImage" type="string">Override OG image URL (public, HTTPS).</ParamField>
<ParamField body="favicon" type="string">Override favicon URL.</ParamField>

<ParamField body="fallbackUrl" type="string">
  Stored on the record as a planned fallback destination. The public redirect does not currently route to it.
</ParamField>

## Example request

<RequestExample>

```bash
curl -X POST https://api.linkutm.com/api/v1/links \
  -H "Authorization: Bearer $TOKEN" \
  -H "x-workspace-id: client-globex" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com/q2-launch",
    "title": "Q2 Launch - Google Search - Brand",
    "customSlug": "q2-launch",
    "utmSource": "google",
    "utmMedium": "cpc",
    "utmCampaign": "q2_launch",
    "utmContent": "headline_a",
    "tagIds": ["tag-uuid-1", "tag-uuid-2"]
  }'
```

</RequestExample>

## Errors

| Code | When |
|---|---|
| `400` | UTM rule violation, invalid domain, validation failure |
| `403` | Workspace link maximum reached, or password-protected link maximum reached |
| `404` | Workspace not found |
| `409` | `customSlug` already exists globally |

## Behavior notes

- UTMs are processed by [workspace UTM rules](/utm-rules/overview) before save (lowercase, space replacement, conditional logic). The stored values may differ from what you sent.
- If `domainId` is omitted, the workspace's default domain is used.
- `tagIds` are attached **after** the link is created, in a separate join-table insert. If any tag UUID doesn't exist or belongs to a different workspace, the join insert fails with a foreign-key error - but the link is already saved at that point. Validate `tagIds` before posting (e.g., `GET /tags`) to avoid orphaned links with no tags.
- Link usage counter is incremented async - your response returns immediately.
- At 80% of the workspace's link maximum, the owner gets a warning email.
- `qrCodeEnabled` is set by the server, not the request body. The link gets `qrCodeEnabled: true` when the workspace is within its QR code quota, and `false` once the quota is reached. It is returned on the link record but cannot be supplied on create.
