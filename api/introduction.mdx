## Overview

The linkutm API is REST + JSON. All endpoints live under `/api/v1`. JSON in, JSON out. Standard HTTP status codes for errors.

## Base URL

```
https://api.linkutm.com/api/v1
```

For self-hosted or staging environments, replace the host accordingly.

## Authentication

All endpoints require a JWT bearer token. Tokens are issued by the auth flow (login response) and identify the user.


```bash
curl https://api.linkutm.com/api/v1/links \
  -H "Authorization: Bearer <jwt_token>" \
  -H "x-workspace-id: <workspace_uuid_or_slug>"
```

<Note>
**Workspace context.** Most endpoints require an `x-workspace-id` header carrying a workspace UUID or slug. Without it, the call returns `404` or `400`.
</Note>

### API keys

Workspaces can also generate long-lived API keys (format `lk_live_<hex>`) for server-to-server usage. Manage them under **Settings → Developer → API keys** in the dashboard.

<Warning>
The current public surface authenticates via JWT bearer tokens. If you're integrating from a backend, exchange your credentials for a JWT via the auth endpoint. API keys exist for management but the auth strategy is JWT-bearer end-to-end.
</Warning>

## Required headers

| Header | Required | Notes |
|---|---|---|
| `Authorization: Bearer <token>` | Yes | JWT token |
| `x-workspace-id: <uuid_or_slug>` | Yes for workspace-scoped endpoints | UUID or slug |
| `Content-Type: application/json` | For POST/PATCH | |

## Versioning

The API is at `v1`. Breaking changes go behind a new version (`v2`). Within `v1`, only additive changes (new optional fields, new endpoints) are made.

## Quick start

<Steps>
  <Step title="Get a token">
    Log in via the auth endpoint or copy a JWT from your browser session.
  </Step>
  <Step title="Find your workspace">
    `GET /workspaces` returns the workspaces you belong to. Note the `id` (UUID) or `slug`.
  </Step>
  <Step title="Create a link">
    ```bash
    curl -X POST https://api.linkutm.com/api/v1/links \
      -H "Authorization: Bearer $TOKEN" \
      -H "x-workspace-id: $WORKSPACE_SLUG" \
      -H "Content-Type: application/json" \
      -d '{
        "url": "https://example.com/landing",
        "utmSource": "google",
        "utmMedium": "cpc",
        "utmCampaign": "q2_launch",
        "customSlug": "q2-launch"
      }'
    ```

    Response: `201 Created` with the full link object.
  </Step>
</Steps>
