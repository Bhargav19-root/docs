# API Documentation

Complete reference for the UTM Builder API endpoints and usage.

## Getting Started

### Authentication
```bash
# API Key Authentication
curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.utmbuilder.com/v1/links
```

### Base URL
- Production: `https://api.utmbuilder.com/v1`
- Staging: `https://api-staging.utmbuilder.com/v1`

## API Concepts

### Rate Limits
- 1000 requests per minute
- 10,000 requests per day
- Burst limits apply
- Rate limit headers

### Response Format
```json
{
  "status": "success",
  "data": {},
  "meta": {
    "page": 1,
    "total": 100
  }
}
```

## Error Handling

### Error Codes
- 400: Bad Request
- 401: Unauthorized
- 403: Forbidden
- 404: Not Found
- 429: Too Many Requests

### Error Response
```json
{
  "status": "error",
  "error": {
    "code": "invalid_request",
    "message": "Invalid parameters"
  }
}
```

## Best Practices

### Implementation
- Use HTTPS
- Handle rate limits
- Implement retry logic
- Log responses

### Security
- Secure API keys
- Use webhooks
- Monitor usage
- Regular audits
