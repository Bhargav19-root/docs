## Delete a single link

```
DELETE /api/v1/links/:id
```

### Headers

| Header | Required |
|---|---|
| `Authorization: Bearer <jwt>` | Yes |

### Example

<RequestExample>

```bash
curl -X DELETE https://api.linkutm.com/api/v1/links/5f3b2a1c-4d6e-4a8b-9c0d-1e2f3a4b5c6d \
  -H "Authorization: Bearer $TOKEN"
```

</RequestExample>

### Behavior

- Cascades click history (`link_clicks` rows are deleted with the link).
- Decrements workspace link usage (only if the link wasn't already archived - archived links are excluded from active count).
- Sends a notification email to workspace members who have `link_deleted` notifications enabled. Async.

### Errors

| Code | When |
|---|---|
| `404` | Link not found |

## Bulk delete

```
POST /api/v1/links/bulk-delete
```

### Body

```json
{
  "ids": [
    "5f3b2a1c-4d6e-4a8b-9c0d-1e2f3a4b5c6d",
    "6a4c3b2d-5e7f-5b9c-ad1e-2f3a4b5c6d7e"
  ]
}
```

### Response

```json
{ "count": 3 }
```

Same cascade semantics as single delete. One consolidated email per workspace.

## Bulk archive / unarchive

```
POST /api/v1/links/bulk-archive
```

### Body

```json
{ "ids": ["uuid-1", "uuid-2"], "archived": true }
```

Setting `archived: true` removes the links from default list views and stops them from resolving (returns `404` on visit). Set `archived: false` to restore.

Only links whose state actually changes contribute to the usage delta - re-archiving an already-archived link is a no-op for the usage counter.

## Bulk tag

```
POST /api/v1/links/bulk-tag
```

### Body

```json
{
  "ids": ["link-uuid-1", "link-uuid-2"],
  "tagIds": ["tag-uuid-1", "tag-uuid-2"],
  "mode": "add"
}
```

| `mode` | Behavior |
|---|---|
| `add` (default) | Append the tags without removing existing ones. Duplicates skipped. |
| `set` | Replace the entire tag set on each link with the provided list. |

<Warning>
`mode: "set"` deletes all existing tags on the listed links before adding the new set. Destructive - use `add` unless replacement is intended.
</Warning>

## Check short code availability

```
POST /api/v1/links/check-shortcodes
```

Pre-flight check for slug conflicts before a bulk create or import. Short codes are globally unique.

### Body

```json
{ "shortCodes": ["q2-launch", "spring-sale", "promo-2026"] }
```

### Response

```json
{
  "conflicts": {
    "q2-launch": true,
    "spring-sale": false,
    "promo-2026": false
  }
}
```

`true` means the short code is already taken - pick a different one for that link.

## Bulk create

```
POST /api/v1/links/bulk-create
```

Create many links in one request. Useful for migrations and bulk imports.

### Body

```json
{
  "links": [
    {
      "url": "https://example.com/q2-launch",
      "shortCode": "q2-launch",
      "title": "Q2 Launch",
      "utmSource": "google",
      "utmMedium": "cpc",
      "utmCampaign": "q2_launch"
    },
    {
      "url": "https://example.com/spring-sale",
      "title": "Spring Sale"
    }
  ]
}
```

Per-link fields: `url` (required), `shortCode`, `title`, `description`, `utmSource`, `utmMedium`, `utmCampaign`, `utmTerm`, `utmContent`, `favicon`, `ogImage`. Omit `shortCode` to auto-generate one.

<Note>
Run `POST /links/check-shortcodes` first to confirm none of your `shortCode` values are taken. Bulk create counts each link against the workspace link maximum.
</Note>

## Import from another shortener

Imports are a two-step flow: **fetch** pulls links from the third-party provider using your credentials for that provider, then **import** writes the returned links into the workspace. This lets you review or filter the fetched set before committing it.

### Short.io

```
POST /api/v1/links/import/shortio/fetch
POST /api/v1/links/import/shortio
```

```json
// fetch
{ "apiKey": "<shortio_api_key>" }

// import
{ "links": [ /* links returned by fetch */ ] }
```

### Bitly

```
POST /api/v1/links/import/bitly/fetch
POST /api/v1/links/import/bitly
```

```json
// fetch
{ "accessToken": "<bitly_access_token>" }

// import
{ "links": [ /* links returned by fetch */ ] }
```

### Rebrandly

```
POST /api/v1/links/import/rebrandly/fetch
POST /api/v1/links/import/rebrandly
```

```json
// fetch
{ "apiKey": "<rebrandly_api_key>" }

// import
{
  "links": [ /* links returned by fetch */ ],
  "fieldMapping": {
    "destination": "destinationUrl",
    "title": "title",
    "slashtag": "slashtag",
    "createdAt": "createdAt"
  }
}
```

`fieldMapping` is optional and lets you remap Rebrandly's field names onto linkutm's `destination`, `title`, `slashtag`, and `createdAt`.

## Notify on import completion

```
POST /api/v1/links/notify-import
```

Triggers an import-completion notification email to the workspace.

### Body

```json
{ "count": 142, "source": "bitly" }
```

`source` is optional. Call this after a bulk import or `bulk-create` finishes.
