Full reference for the `Link` resource. Use this when implementing a typed client.

## Identifiers

| Field | Type | Notes |
|---|---|---|
| `id` | string (UUID) | Server-generated. Stable. |
| `shortCode` | string | Unique globally. 3–15 chars typical, max DB column size 255. |
| `workspaceId` | string (UUID) | Set on creation; immutable. |
| `creatorId` | string (UUID) | The user who created the link. Set on creation. |
| `domainId` | string (UUID) \| null | Which short domain the link uses. |
| `folderId` | string (UUID) \| null | Containing folder, or `null` for unfiled. |

## Behavior controls

| Field | Type | Enforced at redirect? | Notes |
|---|---|---|---|
| `expiresAt` | timestamp \| null | Yes | UTC. Past it → `410 Gone`. |
| `archived` | boolean | Yes | `true` → `404`, hidden from default list views. |
| `password` | string \| null | No | Stored on record; not currently gated by the redirect. |
| `clickLimit` | integer \| null | No | Recorded; not a hard-stop. |
| `fallbackUrl` | string \| null | No | Stored only. |
| `iosUrl` | string \| null | No | Stored only. |
| `androidUrl` | string \| null | No | Stored only. |
| `geoTargets` | object \| null | No | Stored only. |

| Field | Type | Notes |
|---|---|---|

## Relationships

| Field | Type | Notes |
|---|---|---|
| `tags` | `LinkTag[]` | Join records to `Tag` resources. |
| `pixels` | `LinkPixel[]` | Tracking pixels attached to this link. |
| `analytics` | `LinkClick[]` | Click history (only via analytics endpoints, not on the link object). |

## Constraints

- `shortCode`: globally unique. System paths (`api`, `health`, `docs`, `favicon.ico`) won't resolve through the redirect — avoid these as slugs.
- `domain`: must be `verified=true` and `enabled=true` and belong to the same workspace.
- `password`: counts against the workspace's password-protected link maximum (independent of total link maximum).
- All UTM string fields are subject to workspace [UTM Rules](/utm-rules/overview) on save.

## Indexes (informational)

The Postgres table is indexed on `workspaceId`, `shortCode`, `createdAt`, `folderId`. Filter on these for efficient queries.
